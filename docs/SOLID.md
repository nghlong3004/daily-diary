# SOLID

1. S - Single Responsibility Principle
2. O - Open/Closed Principle
3. L - Liskov Substitution Principle
4. I - Interface Segregatio Principle
5. D - Dependency Inversion Principle

## S-Single Responsibility Principle (SRP)
Mỗi class chỉ nên có một lý do để thay đổi

## O - Open/Closed Principle (OCD)
Class nên mở để mở rộng, nhưng đóng để sửa đổi

## L - Liskov Substitution Principle (LSP)
Có thể dùng object của class con thay cho class cha mà không làm hỏng logic chường trình

## I - Interface Segregation Principle (LSP)
Không nên ép client phụ thuộc vào những method mà nó không dùng.

## D - Dpendency Inversion Principle (DIP)
High - level module không nên phụ thuộc trực tiếp vào low-level module, mà cả hai nên phụ thuộc vào abstraction (interface)
